## Making progress

Toward the end of that semester, the teacher gave some of us access to a couple of advanced features of the HP 2000F. One of those features was access to the Fortran compiler.

Compiling and running those programs was done on a scheduled basis. You would write your code and submit a job to have it compiled and run. Then, at some later time, they would run all the jobs and send the output back to the submitter. (I never really learned how that worked.) Because of the load that those jobs put on the system, they were only run in the evening, which meant you were lucky to get one turn-around a day.

If I remember correctly, it wasn't even a full Fortran system - but again, the details escaped me.

I played with it a little bit, but found most of the restrictions too limiting.

That's when he said that they (the school) had a couple of "research accounts" on the University of Michigan computer system - but we had to be _really_ careful with how we used them.

UofM used to rent out time on their mainframe to anyone - that is, anyone willing to pay. And they charged for **everything**. You would get charged for every minute you were dialed in, even if you weren't doing anything. You would get charged for every CPU minute used. You would be charged for disk space usage. You'd get charged for virtual memory usage. You'd get charged for every sheet of paper printed on their printers _and_ for every line printed. You'd get charged for tape mounts. Basically, if there was a resource they could measure, you'd get charged for using it.

And they had different rate schedules based on time-of-day. Running programs in the evening was cheaper that during the day - but running overnight was the cheapest of all.

So we were given a budget. If we kept the account charges below a certain amount per month, there wouldn't be any problem. Run up the charges and you would lose access.

I jumped all over that opportunity. A **real** mainframe to work on!
